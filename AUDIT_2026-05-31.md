# RBMN-Storyboard_App — Full Code Audit
**Date:** 2026-05-31
**Version audited:** 1.6.3 (per `VERSION`)

> **Historical snapshot.** Most CRIT and HIGH items in this report were addressed in **v1.7.0** — see [`CHANGELOG.md`](CHANGELOG.md) for what shipped. Kept for reference / future audits.
**Scope:** Backend services & API, frontend, docs vs code, type-check.
**Method:** Four parallel read-only investigations, plus a static type-check (`tsc --noEmit` passed clean).

This audit was conducted after the two video-dispatch fixes landed earlier this session (wall-clock WS completion poll, Auto Gen modal de-duplication). The findings below are *latent* issues uncovered while looking for similar patterns elsewhere — none of them are user-reported, but several would bite on the next edge case.

---

## TL;DR — Top issues to fix first

1. **`PUT /scenes/reorder` is shadowed by `PUT /{scene_id}`** — scene drag-and-drop reordering returns 422 silently. **(C-1)**
2. **`GET /assets/generated` is shadowed by `GET /{asset_id}`** — the generated-assets list endpoint is unreachable. **(C-2)**
3. **Many `asyncio.create_task(...)` calls without storing the task reference** — the entire auto-gen pipeline can be garbage-collected mid-run. **(C-3, H-2 in API audit)**
4. **Restart cancels in-flight ComfyUI prompts** — common scenario; loses expensive renders. **(M-7 in services audit)**
5. **Demucs subprocess has no timeout** — a wedged Demucs hangs the audio pipeline until app restart. **(C-2 in services audit)**
6. **`Scene.workflow_snapshot` silently stripped by `SceneResponse`** — backend has it, frontend reads it, API response drops it. **(C-4 in API audit)**
7. **`JobResponse` defined twice** in `api/jobs.py` and `api/export.py` — silent OpenAPI schema overwrite (gotcha #5 regression). **(C-2 in API audit)**
8. **Whole-store Zustand subscriptions in 13+ components** — every SSE `job_progress` re-renders huge subtrees. **(H-1…H-7 in frontend audit)**
9. **Path traversal risk in `/api/files/{path}`** — `startswith` without separator guard. **(C-5 in API audit)**
10. **Asset upload reads file fully into memory, no size cap** — 10 GB upload crashes the process. **(C-6 in API audit)**
11. **README version is 1.4.0; actual is 1.6.3.** Custom-node list missing 6 packs that the shipped workflows require. New users will fail their first generation.

## CRITICAL

### CRIT-1 — `PUT /scenes/reorder` shadowed by `PUT /{scene_id}`
- **File:** `backend/api/scenes.py:443` (param route) vs `:591` (`/reorder`)
- **Impact:** Scene drag-and-drop reordering 422s silently. The path literal `"reorder"` fails UUID validation.
- **Fix:** Move `@router.put("/reorder")` above the `/{scene_id}` route. Cross-check every other module — `api/jobs.py` already orders `/stream` and `/purge` before `/{job_id}`, so this is an isolated regression.

### CRIT-2 — `GET /assets/generated` shadowed by `GET /{asset_id}`
- **File:** `backend/api/assets.py:216` (`/{asset_id}`) vs `:578` (`/generated`)
- **Impact:** The list-generated-assets endpoint is unreachable. Whatever frontend feature consumes it (asset manager / gallery) silently returns 422.
- **Fix:** Reorder.

### CRIT-3 — Duplicate Pydantic class `JobResponse`
- **Files:** `backend/api/jobs.py:24` (13 fields) AND `backend/api/export.py:96` (5 fields)
- **Impact:** Both classes share an OpenAPI component name. Whichever loads second overwrites the schema. Documented in gotcha #5; this regression is exactly the pattern that originally caused the rename to `GenerationJobResponse` in another router.
- **Fix:** Rename `export.py`'s class to `ExportJobResponse`.

### CRIT-4 — `Scene.workflow_snapshot` dropped from `SceneResponse`
- **Files:** `backend/database/models.py:115` (DB has it) vs `backend/api/scenes.py:112-129` (`SceneResponse` doesn't expose it). Frontend `types/index.ts:53` reads it.
- **Impact:** Gotcha #54 regression — silent data loss across every GET/PUT scene.
- **Fix:** Add `workflow_snapshot: Optional[dict] = None` to `SceneResponse`.

### CRIT-5 — Path traversal in `/api/files/{path}` prefix check
- **File:** `backend/api/files.py:31, 69`
- **Issue:** `str(full_path).startswith(str(project_dir_resolved))` lacks a separator-boundary guard. A sibling like `/foo/bar2` matches prefix `/foo/bar`.
- **Fix:** Use `full_path.is_relative_to(project_dir_resolved)` (Python 3.9+) or append `os.sep` before the `startswith` check.

### CRIT-6 — Asset upload reads file fully into memory, no size cap
- **File:** `backend/api/assets.py:98`
- **Issue:** `await file.read()` slurps the whole upload. A 10 GB drag-drop crashes the process.
- **Fix:** Stream to disk with `aiofiles.open` in chunks and enforce a `max_upload_bytes` from settings.

### CRIT-7 — Demucs subprocess has no timeout (can hang forever)
- **File:** `backend/services/audio/analysis.py:284`
- **Issue:** `Popen` + `process.wait()` with no timeout. If Demucs wedges, the request thread is stuck until app restart.
- **Fix:** `process.wait(timeout=1800)` + `process.kill()` on `TimeoutExpired`.

### CRIT-8 — Many `asyncio.create_task(...)` calls without storing the reference (GC risk)
- **Files:** `backend/api/generation.py:2131` (the auto-gen entry point!), `:2190`, `backend/api/export.py:884`, `:1099`, `backend/api/batch_runs.py:164`, `backend/api/batch.py:311, 343, 373, 503, 686, 715, 752, 997, 1213`. Roughly **15+ instances**.
- **Issue:** Python's event loop holds only a weak reference; the task can be GC'd mid-flight. Exceptions are silenced.
- **Impact:** The single biggest risk is `_run_sequential_auto_gen` — the user's entire pipeline can vanish under memory pressure with no error.
- **Fix:** Module-level `set[asyncio.Task]` + `task.add_done_callback(_tasks.discard)`. Mirror the `_active_tasks` pattern already used in `services/jobs/dispatcher.py`.

## HIGH

### HIGH-1 — Worker `in_flight` drift on retry fast path
- **File:** `backend/services/jobs/dispatcher.py:475-519` (retry skips `select_worker(reserve=True)`) vs `services/comfyui/dispatcher.py:606-608` (finally always decrements)
- **Impact:** Each retry decrements without an increment. Counter monotonically drifts toward zero; eventually `select_worker` over-schedules a busy worker → VRAM contention / OOM.
- **Fix:** Increment `worker.in_flight` before `_monitor_and_complete` on the retry path.

### HIGH-2 — Restart cancels in-flight ComfyUI prompts
- **File:** `backend/services/jobs/queue.py:257-288` (`recover_running_jobs`)
- **Issue:** All PENDING+RUNNING jobs are CANCELLED on every startup. If ComfyUI is still rendering, the result lands in `/history` but no asset record is created.
- **Impact:** Restarting the backend during a long LTX render loses the render.
- **Fix:** For RUNNING jobs with `prompt_id` + `worker_url`, reconnect via the existing fast-path in `_process_job`. Only cancel jobs with no in-flight prompt.

### HIGH-3 — Sync `requests.get()` inside async routes (event loop blocking)
- **File:** `backend/api/settings.py:840, 846, 906, 916, 925, 934` (test-whisper endpoints)
- **Fix:** `await asyncio.to_thread(...)` or switch to `httpx.AsyncClient`.

### HIGH-4 — Sync `open()` in async routes
- **Files:** `backend/api/assets.py:122`, `backend/api/timeline.py:324, 508, 884, 2158, 2215`, `backend/api/batch.py:160, 963`, `backend/api/backing_tracks.py:160`
- **Fix:** `aiofiles` or `asyncio.to_thread`.

### HIGH-5 — `mux_audio` in narration export bypasses `_run_ffmpeg`
- **File:** `backend/services/video/assembly.py:1282-1297`
- **Issue:** Raw `subprocess.run(..., timeout=120)` with no partial-output cleanup. On `TimeoutExpired` leaves a zero-byte muxed file and silently moves on. Long narration videos may export with missing-audio chunks.
- **Fix:** Route through `_run_ffmpeg`; raise on timeout.

### HIGH-6 — `prompt_id` missing from `JobResponse`
- **File:** `backend/api/jobs.py:24-43` (model omits `prompt_id`) vs `models.py:244`
- **Impact:** Frontend can't show the ComfyUI prompt id; debugging stuck jobs requires log diving.
- **Fix:** Add `prompt_id: Optional[str] = None`.

### HIGH-7 — `cancel_job` allow-list includes DONE/FAILED
- **File:** `backend/api/jobs.py:294-310`
- **Issue:** A stale cancel can flip a DONE job to CANCELLED and emit a misleading SSE event.
- **Fix:** Restrict to `[PENDING, RUNNING]`, return 409 otherwise.

### HIGH-8 — `datetime.now()` (local time) in some API responses
- **Files:** `backend/api/timeline.py:57, 62` and `backend/services/llm/prompt_enhancer.py:35, 38`
- **Issue:** Rest of codebase uses `datetime.utcnow()`. Frontend normalizes "no-Z = UTC" (gotcha #46). Local-time strings will display shifted by TZ offset.
- **Fix:** Replace with `datetime.utcnow()`.

### HIGH-9 — `BackingTrack` missing cascade → DELETE Project fails
- **File:** `backend/database/models.py:277-294` (no `cascade_delete=True`) vs `database.py:51` (`PRAGMA foreign_keys=ON`)
- **Impact:** Deleting any project with backing tracks raises a FK violation.
- **Fix:** Add `Relationship(back_populates=..., cascade_delete=True)` on Project, or `ondelete="CASCADE"` at column level.

### HIGH-10 — Whole-store Zustand subscriptions cause re-render storms
Affected files (13 total) — replace `useAppStore()` destructure with per-field selectors:
- `AppLayout.tsx:91-92` (root!), `:1308` (SceneList)
- `Timeline.tsx:21-27`
- `SceneEditor.tsx:758` (~3400 lines, mounted continuously)
- `WaveformDisplay.tsx:32`
- `useBackingTrackPlayer.ts:54` (called 30x/sec while playing)
- `BackingTrackTimeline.tsx`, `VideoFlowPanel.tsx:21`, `AssetManager.tsx:39`, `AssetManageModal.tsx:69`, `AudioSetup.tsx:88`, `GenerationPanel.tsx:41`, `SectionMarkers.tsx:18`, `VideoPreview.tsx:64`, `ReferenceSelector.tsx:65`, `CharacterCreatorModal.tsx:106`

**Why it matters:** SSE `job_progress` events fire many times per second during batches. Each one updates `jobs` in the store. Whole-store subscribers re-render the entire AppLayout tree on every tick. This is the single biggest UI-perf issue.

**Fix template:**
```ts
// Before
const { setProject, setScenes } = useAppStore();
// After
const setProject = useAppStore(s => s.setProject);
const setScenes = useAppStore(s => s.setScenes);
```

### HIGH-11 — README "Required Custom Nodes" is incomplete
The README lists 8 packs but the shipped workflows reference at least 6 more that the user will need:
- `DetailDaemonSamplerNode` (Klein 1REF–5REF + T2I)
- `FastFilmGrain`, `FastLaplacianSharpen` (Klein T2I)
- `ComfySwitchNode` (Klein)
- `LTXVFirstLastFrameControl_TTP` (LTX FFLF — TTPlanet pack)
- `ResizeImagesByLongerEdge` (LTX FFLF)
- `TrimAudioDuration` (LTX FFLF)

**Impact:** Fresh users following README will get "missing node" errors on first generation. The dispatcher does auto-remove non-essential nodes (gotcha #29), but these are *essential*.

**Fix:** Update the README table.

## MEDIUM

### MED-1 — `JobEventBroadcaster._subscribers` mutated without lock
- **File:** `backend/services/jobs/dispatcher.py:48-81`
- **Issue:** `subscribe/unsubscribe` mutate a plain list while `put_nowait` iterates. The lock attribute is declared but never used. Many concurrent SSE clients → `RuntimeError: list changed during iteration`.
- **Fix:** Iterate over `list(self._subscribers)` snapshot, or actually use the lock.

### MED-2 — `update_project` doesn't bump `updated_at`
- **File:** `backend/api/projects.py:225-292`
- **Impact:** `list_projects` sorts by `updated_at DESC`; edits don't move projects to the top of the list.
- **Fix:** Set `project.updated_at = datetime.utcnow()` before commit.

### MED-3 — `BatchRun.project_id` is plain `str` (no FK)
- **File:** `backend/database/models.py:416`
- **Impact:** Orphan batch runs survive project deletion. Either add FK + migration, or document as intentional.

### MED-4 — Missing-Z timestamp normalization in 5 frontend files
Frontend audit found 5 locations using `new Date(ts)` without first appending `Z`:
- `BatchesDashboard.tsx:34` (formatDate)
- `AppLayout.tsx:2405` (ExportGalleryModal formatDate)
- `AssetManageModal.tsx:99, 179`
- `GenerationPanel.tsx:20-21, 231` — live elapsed counter is **off by TZ offset** while jobs run
- `useJobEvents.ts:149` — elapsed_ms in last-completed-asset is off by TZ offset
- `SceneEditor.tsx:195, 408, 411, 2702, 3276`
- `AssetGeneratorModal.tsx:1028`, `ProjectList.tsx:232`, `SettingsPage.tsx:459-460`

**Fix template:** Apply the existing normalization helper used in `BatchRunDetail.tsx`.

### MED-5 — `recover_running_jobs` doesn't reconnect to live prompts
See HIGH-2; the design itself is medium-priority to retrofit. Quick patch: if RUNNING + `prompt_id` + `worker_url`, skip the cancel.

### MED-6 — Migration resets `color_correction_enabled` to 0 on every startup
- **File:** `backend/database/database.py:238-239`
- **Issue:** `UPDATE color_correction_enabled = 0 WHERE = 1` runs every boot. The user's choice never persists.
- **Fix:** Guard with a sentinel migration row, or check a `migrations_applied` flag.

### MED-7 — CORS `allow_origins=["*"]` + `allow_credentials=True`
- **File:** `backend/main.py:177-183`
- **Browsers block this combination** so it's not exploitable today, but if `network_access=True` and a proxy strips the rejection, the surface area opens. Restrict to `http://localhost:*` etc.

### MED-8 — N+1 query in `list_projects`
- **File:** `backend/api/projects.py:130-155` — 2 extra SELECTs per project (scenes_count, assets_count)
- **Fix:** `func.count` + JOIN or `selectinload`.

### MED-9 — `frontend/ARCHITECTURE.md` and `frontend/README.md` are stale
Both stuck at v1.0-era component lists. Missing: AssetGenerator, AudioSetup, BackingTrackTimeline, BatchMode, BatchPreviewPIP, ConceptPanel, ErrorBoundary, VideoFlowPanel. SceneEditor tab list is wrong (missing FF/LF sub-tabs, Tools, Image Movement, Prompt).

### MED-10 — `CharacterCreatorModal.tsx:194-197` interval leak
```ts
const poll = setInterval(() => refetchVersions(), 3000);
setTimeout(() => clearInterval(poll), 60_000);
```
No ref, no `useEffect` cleanup. If the modal unmounts before 60s, polling continues.

### MED-11 — `_resolve_v2v_last_frame_image` in hot path with no time cap
- **File:** `backend/services/jobs/dispatcher.py:2756-2790`
- **Issue:** Synchronously runs up to 4 ffmpeg extract attempts inside `_build_builtin_workflow`. No cumulative timeout.
- **Fix:** Cap total time; bail to `chosen_image_path` fast.
